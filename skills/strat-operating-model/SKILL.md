# ターゲット・オペレーティングモデル

## 概要

ターゲット・オペレーティングモデル（TOM）フレームワークは、戦略の意図を組織の設計図に変換するものです。どのようなケイパビリティが必要か、業務プロセスがどのように流れるか、誰が何を担うか、そして人材・プロセス・技術というイネーブラーが何を満たすべきかを具体的に示します。このフレームワークが示す核心的な知見は、「戦略が失敗するのは方向性の誤りではなく、実行するための組織が整っていないことが原因である」という点です。

## 使用する場面

- 戦略の方向性が決まり、それを実行するための組織をどのように整えるかを検討するとき。
- 合併・統合、デジタル変革、または事業範囲の大幅な変更により、既存モデルが不十分になっているとき。
- 責任の所在の曖昧さ、業務の重複、意思決定の遅さが顕在化し、組織構造とガバナンスを見直す必要があるとき。
- **使用しない場面：** 戦略的な選択肢がまだ固まっていない段階。戦略が不明確なままオペレーティングモデルを設計すると、誤った方向への最適化につながります。戦略の検討が先決な場合は `strat-strategic-options` を先に実施してください。また、安定した組織内での個別の業務改善には、プロセスマッピングが適しています。

## 必要な情報

以下の情報を事前に収集してください。不足している項目があれば、進める前に貴社にご確認ください。数値や組織情報を勝手に補完してはなりません。

- **戦略の概要：** ターゲットとする顧客セグメント、提供する価値、主要な差別化要素。
- **現状のオペレーティングモデル（As-Is）：** 組織構造、主要な業務プロセス、ガバナンス体制、主要なシステム、機能別の人員数。貴社にご確認いただくか、不明な項目として明示してください。
- **現状の課題：** 現行モデルで問題となっているポイント（スピード、コスト、品質、責任の所在の曖昧さ等）。
- **設計上の制約：** 予算規模、スケジュール、法規制・コンプライアンス要件、労使協定、拠点の地理的条件。
- **意思決定の時間軸：** 12か月での再設計なのか、3年かけた変革なのか。目指す水準によって、許容できるギャップの大きさが変わります。

## 進め方

1. **戦略から必要なケイパビリティを導出する。** 価値提供のために貴社が特に優れていなければならない事項を列挙します。「競争優位を生むケイパビリティ」（差別化の源泉となるもの）と「事業遂行に必要な基本ケイパビリティ」（事業運営に欠かせないが差別化につながらないもの）を分けて整理します。

2. **ターゲット・オペレーティングモデルを5つの次元で設計する。** 各次元について、あるべき姿を記述します。
   - **組織構造：** 必要なケイパビリティを最も効果的に発揮できる、業務のまとまりと報告ラインの設計。
   - **業務プロセス：** 主要な一気通貫の業務フロー、オーナーシップの所在、それを管理するKPI。
   - **ガバナンス：** 意思決定権限（誰が何をどの階層で決めるか）、エスカレーションの経路、横断的な調整の場。
   - **人材：** 必要な役割・スキル・人員構成。現在存在しない役割はここで明示します。
   - **技術：** ターゲット業務プロセスを支えるシステムやデータの流れ。既存システムの制約も確認します。

3. **ターゲットと現状を比較する。** 5つの次元それぞれについて、現状をターゲットと並べて差異を確認します。差異はギャップであり、現状の失敗ではないという姿勢で、客観的に記述します。

4. **ギャップを特定し、分類する。** 各ギャップをタイプ（組織構造・業務プロセス・ガバナンス・人材/スキル・技術）と深刻度（戦略実行を阻害するもの vs 効率を低下させるもの）で分類します。深刻度が高く、かつ解決に時間がかかるギャップは移行スケジュール全体を左右するため、特に注目します。

5. **移行計画を概略する。** 現状からターゲットへのフェーズ別の道筋を提示します。何をどの順番で変えるか、先決すべき依存関係、モデルが定着するまでの暫定措置を整理します。移行の順序に関する前提は、確定とする前に必ず貴社にご確認ください。

## 成果物

このスキルは、以下の構成のスタンドアロン分析（マークダウン形式）を生成します。

```
# オペレーティングモデル — <対象>
## 必要なケイパビリティ
## ターゲット・オペレーティングモデル
## ギャップ
## 移行計画
## 要点
## 確認事項
```

冒頭のタイトルは出力契約に従い `# オペレーティングモデル — <対象>` の形にします。`<対象>` は具体的な案件テーマです（例：「オペレーティングモデル — ホライゾン物流株式会社 デジタル変革」）。

`## 必要なケイパビリティ` では、戦略から導出した差別化ケイパビリティと基本ケイパビリティを列挙します。

`## ターゲット・オペレーティングモデル` では、組織構造・業務プロセス・ガバナンス・人材・技術の各次元のあるべき姿を記述します。

`## ギャップ` では、現状とターゲットモデルの差異を、タイプ別・深刻度別に整理します。

`## 移行計画` では、フェーズ別の順序、依存関係、暫定措置を概略します。

`## 要点` では、最も重要な結論を箇条書きでまとめます。

`## 確認事項` では、確認が必要な数値・前提・判断、またはモデルの最終化に先立って必要な意思決定事項を列挙します。

## 例

**例 — 架空の中堅物流企業「ホライゾン物流株式会社」（例示・確認前）**

ホライゾン物流の新戦略は、スポット貨物の手配仲介から、中堅製造業者向けの長期マネージド物流契約への転換です。このスキルの適用イメージを以下に示します。

**必要なケイパビリティ（ステップ1）**

- 差別化ケイパビリティ：顧客別の需要予測、大規模なキャリア関係管理、異常時の先手対応、契約収益（P&L）管理。
- 基本ケイパビリティ：積載予約、請求処理、コンプライアンス対応、基本的な貨物追跡。

**ターゲット・オペレーティングモデル — 抜粋（ステップ2）**

- 組織構造：長期契約を専任で担う「顧客ソリューション部門」を新設し、既存のスポット対応部門と明確に分離する。
- ガバナンス：顧客担当ディレクター・オペレーションリード・財務が週次で契約収益を共同管理する「アカウントレビュー会議」を設置する（現時点では存在しない）。
- 人材：「顧客担当ディレクター」（現在この役割は未設置）と「需要計画担当」（現在2名。適正人員数は確認事項）。

**ギャップ（ステップ3〜4）**

| 次元 | ギャップ | 深刻度 |
|---|---|---|
| 組織構造 | 顧客ソリューション部門が未設置 | 阻害（Blocking） |
| ガバナンス | 契約P&L管理の運営体制が未設置 | 阻害（Blocking） |
| 人材 | 顧客担当ディレクター職が未定義 | 阻害（Blocking） |
| 技術 | 需要予測ツールが未整備 | 高（解決に時間を要する） |

※ 上記の深刻度は例示です。正式な評価は貴社との確認を経て確定します（確認前）。

**移行計画（ステップ5）**

フェーズ1（1〜3か月）：顧客ソリューション部門を暫定体制で発足。顧客担当ディレクターの役割定義を実施。
フェーズ2（4〜9か月）：顧客担当ディレクターの採用・育成。アカウントレビュー会議のガバナンスを確立。需要予測ツールの選定を開始。
フェーズ3（10〜18か月）：オペレーティングモデルの全面稼働。スポット対応部門は繁閑対応のオーバーフローとして再配置。

※ 上記の期間・順序は例示です。確定前に貴社にてご確認ください（確認前）。

その後、**要点**と**確認事項**のセクションで、分析の結論と確認が必要な事項（人員数の予算、技術ベンダー候補の承認、スケジュールの実現可能性等）をまとめます。
