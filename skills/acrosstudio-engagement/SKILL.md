# エンゲージメント統括

## 概要

エンゲージメント統括スキルは、コンサルティング案件の全工程を一つのまとまった流れとして実行します。インテーク・戦略分析・成果物作成の各スキルを個別に手動で呼び出すのではなく、一気通貫で走らせます。その中心となる仕組みが、ファイルベースの**エンゲージメント・ワークスペース**です。`engagement/` という作業ディレクトリに、番号付きのマークダウン成果物を順次書き出していきます。各サブスキルは互いに独立したままで、この番号付きファイルを通じてのみ連携します。どのスキルを走らせてどのファイルに出力するかを決めるのはオーケストレーターの役割です。個々のスキルは通常の出力契約（`# <フレームワーク> — <対象>` で始まり `## 要点` と `## 確認事項` で締めくくる分析）をそのまま維持します。スキル間の結合は低く保たれており、不要なステージを省くことは常に有効です。

## 使用する場面

- 複数の分析を一つの成果物へ収束させる必要がある、正式なご提案や多段階のエンゲージメント。
- フレームワークの適切な順序を決めること自体が仕事の一部である場面。インテークの結果をもとに、どの診断・選択・実行・コミュニケーションのスキルをバンド順で走らせるかを判断し、一つの複合ドキュメントにまとめます。
- ある分析で生じた確認事項を、次のステージへ進む前に確認しなければならないエンゲージメント。
- 一部のバンドを省略するエンゲージメント。絞り込んだ分析セット（フレームワーク数本とデッキ）は、バンドを減らした同じパイプラインであり、別のプロセスではありません。

**使用しない場面：**

- 単体のスタンドアロン分析。該当の `strat-*` スキルを直接呼び出し、その出力をそのまま参照してください。
- 分析チェーンを必要としない簡易ドキュメント。`acrosstudio-pptx` を直接呼び出してください。
- すでに存在するエンゲージメントの一ステージだけを再実行する場合。該当のスキルを単独で呼び出し、出力を正しいバンドのファイル名でワークスペースに配置してください。

## 必要な情報

- **エンゲージメントの依頼内容。** 何を達成すべきかを示す利用者の説明。どれほど簡潔でも構いません。これが唯一の必須情報です。
- **既存の事前説明（任意）。** 確認済みの完全なブリーフが書面ですでに存在する場合、インテーク・ステージを省略し、ブリーフを `00` 成果物として直接保存できます。完全なブリーフがない場合は、内容を推測せずインテークを実行してください。

すべてのステージで、貴社の数値を勝手に作成してはなりません。不明な数値は確認事項として利用者へ差し戻し、推測で埋めません。この原則は、依頼内容に数値がない場合も、`strat-*` 分析で生じた場合も、成果物の作成に必要な場合も、等しく適用されます。

## 進め方

1. **スコープを決める。** 「インテーク → 診断 → 選択 → 実行 → コミュニケーション」のフルご提案か、目的に合ったバンドだけを走らせる絞り込み分析か、既存のブリーフや分析をもとにした成果物のみの実行か、いずれのスコープかを判断します。この判断が、実行するバンドとスキルを決めます。

2. **インテークを実行する。** 確認済みのブリーフが存在しない限り、`acrosstudio-intake` を呼び出してその出力を `engagement/00-intake.md` に書き出します。確認済みのブリーフがある場合はそれを `engagement/00-intake.md` として保存し、対話を省略します。

3. **バンドとスキルを選定し、バンド順に実行する。** `reference/workspace-contract.md` のバンド体系に基づき、目的に合ったスキルを選んでバンド順に実行し、各出力をバンドのファイル名で保存します。診断（`10`〜`19`）：`strat-situation-assessment`、`strat-growth-barriers`、`strat-assumption-audit`。選択・採算（`20`〜`29`）：`strat-strategic-options`、`strat-business-case`、`strat-pricing`、`strat-portfolio-review`。実行（`30`〜`39`）：`strat-operating-model`、`strat-initiative-prioritizer`、`strat-transformation-roadmap`。コミュニケーション（`40`〜`49`）：`strat-narrative`、`strat-decision-memo`、`strat-stakeholder-alignment`。バンド内では実行順に番号を振ります（例：`10-situation-assessment.md`、`11-growth-barriers.md`）。

4. **各バンド終了後に確認事項を確認する。** そのバンドで書き出した各成果物の `## 確認事項` セクションを集約し、利用者へ提示します。未確定の数値・スコープ・前提について利用者の確認を待ってから次のバンドへ進みます。パイプラインを止めないために数値を推測で作り上げることはしません。未確認の数値は確認事項のままにします。

5. **成果物を作成する。** 分析がスコープに対して十分な水準に達したら、`acrosstudio-pptx` を呼び出し、`engagement/*.md` をコンテンツソースとして指定します。pptx スキルは存在する成果物を読み込むため、省略したバンドは出力に影響しません。

6. **報告する。** ワークスペースの内容（各 `engagement/NN-*.md` 成果物と生成スキルの対応）、全バンドの `## 確認事項` を集約した確認事項ログ、最終成果物のパスを報告します。この集約ログが利用者の承認対象であり、オーケストレーターが推測で埋めた数値を一切含まないことを確認します。

## 成果物

- 入力されたエンゲージメントの `engagement/` ワークスペース：`engagement/00-intake.md` と、実行した各 `strat-*` スキルの番号付き成果物。各成果物は `## 要点` / `## 確認事項` で締めくくる独立した分析です。
- `acrosstudio-pptx` が生成した最終成果物（`.pptx`、費用や数値が伴う場合は `.xlsx` も）。
- 成果物とスキルの対応と、利用者の確認待ちの確認事項を列挙した簡易報告。
- 最終成果物は `# エンゲージメント統括 — <対象>` をトップレベルタイトルとして開始し、各バンドの貢献は元の `# <フレームワーク> — <対象>` というソースタイトルで表示されます。これによりレビュアーは任意のスライドを生成元の成果物まで追跡できます。

バンド体系の正本、契約ルール、ステップごとの成果物マッピングは `reference/workspace-contract.md` を参照してください。

## 例

**例 — 架空の精密部品メーカー「株式会社ミドリ精工」（例示のみ・確認前の数値は含みません）。**

利用者の依頼：「精密部品メーカーのミドリ精工に成長戦略のご提案をしたい。デッキを作ってほしい。」オーケストレーターはフルご提案のスコープで走り、次のワークスペースを生成します。

- `engagement/00-intake.md` — `acrosstudio-intake` が貴社の文脈・目的・スコープ・制約、および確定済みの数値と未確認の数値を整理します。
- `engagement/10-situation-assessment.md` — `strat-situation-assessment` が現状診断を確立します（診断バンド）。
- `engagement/20-strategic-options.md` — `strat-strategic-options` が成長の選択肢を展開します（選択・採算バンド、実行順 1 番目）。
- `engagement/21-business-case.md` — `strat-business-case` が有力オプションの採算を構築します（同バンド、実行順 2 番目）。
- `engagement/30-transformation-roadmap.md` — `strat-transformation-roadmap` が展開フェーズを計画します（実行バンド）。
- `engagement/40-narrative.md` — `strat-narrative` がご提案の主要メッセージを構築します（コミュニケーションバンド）。

各バンド終了後、オーケストレーターはそのバンドの確認事項（例：ミドリ精工の確定売上高や回収期間の前提。いずれも例示・確認前）を利用者へ提示し、確認を得てから次のバンドへ進みます。数値を推測で作成することはありません。最後に `acrosstudio-pptx` が `engagement/*.md` を読み込み、`midori-seiko-growth-proposal.pptx`（採算ファイル `midori-seiko-business-case.xlsx` を含む）を生成します。オーケストレーターは 6 つの成果物と各生成スキルの対応、成果物のパス、未確認の確認事項を報告します。
