# 文書タイプ別ガイド

Acrosstudio が作成する 3 つの文書カテゴリ（提案書 / パーツ資料 / 打合せ資料）の構成と特徴。

## カテゴリ 1：提案書

### 1.1 フル提案書（9〜10 枚構成）

新規案件で顧客にアプローチする際の主要文書。

#### 標準構成

| # | スライド | 内容 |
|---|---|---|
| 1 | 表紙 | タイトル、Ver、日付、白ロゴ |
| 2 | 背景・課題 | 相談内容の理解、過去の検討経緯、現状の業務フロー |
| 3 | 本提案の方向性 | アプローチの全体像、活用する技術（VLM 等）の説明 |
| 4 | 提案 1 詳細 | 内容、入力、出力、期待される効果 |
| 5 | 提案 2 詳細 | 同上 |
| 6 | 提案 3 詳細 | 同上 |
| 7 | 概算金額感 | 3 案の金額テーブル（概念実証・本開発・年間運用保守） |
| 8 | Appendix | 今後の展開候補（A〜G） |
| 9 | 進め方・次のステップ | 4 STEP の流れ |

#### ファイル名規約
- `{客先英名}_{案件概要}_v{バージョン}.pptx`
- 例：`テクノリペア_AI活用提案_v0.1.pptx`

### 1.2 単一テーマの提案書（単一テーマ提案書サンプル）

複数テーマを並行する案件で、テーマごとに作成。

#### 標準構成（19 枚程度）

| 章 | 内容 |
|---|---|
| **0. 表紙** | テーマ名、Ver、日付 |
| **1. 背景** | 業界背景、課題、過去の検討 |
| **2. 本提案の方向性** | 全体アプローチ、サービス全体像、適用業務 |
| **3. システムの内容** | 入力、AI 処理、出力、効果、業務フロー比較、UI イメージ |
| **4. 実行計画と見積** | プロジェクトスケジュール、実施体制、見積 |

#### ファイル名規約
- `{NN}_{Category}_{ThemeName}_v{バージョン}.pptx`
- 例：`02_Equipment_ProcessAdjustment_v5.pptx`

## カテゴリ 2：パーツ資料

提案書の一部、または提案書とセットで顧客に提示する補助資料。

### 2.1 入出力整理（1 テーマ 1 枚 × 複数）

各テーマで「**何が入って、何が処理されて、何が出てくるか**」 を 1 枚で整理。

#### 構成
- タイトル（章番号 + テーマ名 + 「─ 入力と出力の整理」）
- ユーザー欄（誰が使うか）
- リード文（NAVY_LIGHT 帯、何をするシステムか）
- 3 カラム：入力（CLIENT 青）/ AI 処理（NAVY）/ 出力（MAGENTA）

#### 注意点
- 各カラム 3〜5 項目程度
- 見積依頼文・要件定義をベースに、推測で情報を盛らない
- テーマ間連携は内容で説明（「03 から提供」 ではなく「洗浄品質評価で構築されるもの」）

### 2.2 運用費試算（1 テーマ 1 枚 × 複数）

各テーマの月次・年次のインフラ運用費を、クラウド構成 / オンプレ構成で比較。

#### 構成
- タイトル + 「運用費用の試算」
- 前提（NAVY_LIGHT 帯）：システム本体は売り切り、装置数規模、保管期間など
- 2 カラム：クラウド構成 / オンプレ構成
- 各カラムに内訳 4 項目 + 月額合計 + 年額合計
- 注記：変動要因の説明

#### 注意点
- 「**問合せ対応・ML 再学習は含まず**」 を前提に明示
- 装置数規模・保管期間・バッチ前提を明示
- ROI Excel との数字整合性を取る（複数資料での齟齬を避ける）

### 2.3 ROI 試算 Excel

投資額・効果・運用費から ROI と回収期間を試算するスプレッドシート。

#### 標準構成（シート構成）
- **サマリ**：4 テーマの ROI 一覧
- **前提条件**：評価期間、人件費単価、装置数、保管期間
- **{NN}_{ThemeName}**：各テーマシート

#### 各テーマシートの構成
1. **投資コスト**：要件定義 / 開発 / 検証 など段階ごとの金額
2. **年間運用保守（インフラ費）**：構成タイプ（cloud/onprem）+ 内訳 4 項目 + 月額・年額合計
3. **効果項目**：効果ラベル + 現状値 + 単位 + 改善率 + 年間効果
4. **ROI 試算**：投資総額、年間ネット効果、累計、純利益、ROI、回収期間

#### 変動可能なパラメータ
- **対象装置数**（前提条件シート C9）：装置数連動の効果値・運用費が連動
- **構成タイプ**（各テーマシート C15）：cloud / onprem を選択
- **入力セル**（黄色背景）：投資金額、改善率、効果単価

#### スタイル
- 入力セル：黄色背景（`#FFF2CC`）+ 青字 + 太字
- 計算セル：グレー背景（`#F2F2F2`）
- 結果セル：NAVY 背景 + 白字 + 太字
- ヘッダー行：NAVY 背景 + 白字 + 中央揃え

### 2.4 概算金額感

3 案 × 4 項目（概念実証 / 本開発・年額 / 本開発・累計 / 年間運用保守）の金額テーブル。

#### 構成
- ヘッダー：「概念実証（円・税抜）」「本開発  年額（円）」「本開発  累計 5 年（円）」「運用後  年間運用保守（円）」
- 行：3 案 + 4 テーマ + 注記

## カテゴリ 3：打合せ資料

社内打合せ・客先打合せで使用する文書。

### 3.1 議事録

#### 構成
1. **ヘッダー**：日時、場所、参加者、議題
2. **要点・決定事項**：3〜5 項目で要約
3. **議論内容**：論点ごとに整理
4. **宿題・next action**：誰が・いつまでに・何をするか

#### スタイル
- フォーマル、簡潔
- 5W1H を意識
- 決定事項と未決事項を明確に区別

### 3.2 検討資料

特定の論点に対する検討結果をまとめる資料。

#### 構成
1. **背景・目的**：なぜこの検討をしているか
2. **現状認識・課題**：今の状況と困っていること
3. **検討した選択肢**：複数案を比較（メリデメ表）
4. **推奨案**：どれを選ぶべきか、理由
5. **リスク・留意点**：選択した場合の注意点

#### スタイル
- メリデメ表を多用
- 比較が分かりやすいレイアウト
- 推奨案を明確に

### 3.3 報告書

プロジェクト進捗・調査結果・実証結果などを報告する資料。

#### 構成
1. **サマリ**：1 行で結論
2. **報告内容**（章立て）：詳細
3. **データ・根拠**：図表、数値
4. **結論・next action**：次に何をするか

#### スタイル
- 結論ファースト（サマリで全体像）
- 詳細は章立てで構造化
- データ・根拠を必ず示す

### 3.4 論点整理

複雑な議論を整理し、論点を可視化する資料。

#### 構成
1. **論点一覧**：論点を箇条書きで列挙
2. **各論点の詳細**：背景、現状、選択肢、推奨
3. **論点の関係性**：論点間の依存関係（あれば）
4. **議論の進め方**：どの順序で議論すべきか

#### スタイル
- 論点を網羅的に列挙
- 視覚的な整理（フローチャート、マトリクス）
- 議論をリードするための構成

## 共通事項

### どの文書でも守ること

| 項目 | ルール |
|---|---|
| マスター | 黒ロゴ右上、コピーライト右下を必ず |
| 配色 | カラーパレット内のみ使用 |
| フォント | BIZ UDPGothic / Arial |
| 言葉遣い | 「貴社」「弊社」「担当の方」 で統一 |
| 客先固有名 | 抽象化 |
| 推測ベース情報 | 必ず明示 |
| 視覚確認 | 完成後に PDF 化 → 画像確認 |

### 文書タイプの選び方

| 場面 | 推奨文書タイプ |
|---|---|
| 新規案件のアプローチ | フル提案書（9〜10 枚） |
| 既存案件のテーマ追加 | 単一テーマ提案書 |
| 提案書を補完する詳細 | パーツ資料（入出力整理、運用費、ROI） |
| 社内・客先での議論 | 検討資料 or 論点整理 |
| 進捗・調査結果の共有 | 報告書 |
| 会議の記録 | 議事録 |

## 共通事項 ── 数字・金額の参考レンジ

実案件の蓄積から得た参考値。**絶対値ではなく目安**として活用する。

### 開発フェーズの見積目安（AI 案件）

| フェーズ | 期間 | 金額目安（税抜） | 備考 |
|---|---|---|---|
| 要求整理・提案 | 約 1 ヶ月 | 30〜100 万円 | 提案書作成期間 |
| 要件定義 | 約 2 ヶ月 | 300〜800 万円 | スコープ・仕様確定 |
| 開発 | 約 3〜6 ヶ月 | 1,500〜3,000 万円 | テーマ規模次第 |
| 検証（実環境） | 約 2〜3 ヶ月 | 500〜1,500 万円 | 連携・実環境検証 |

**注**：金額は弊社（中堅 AI 専業）の場合。大手 SI なら +30〜80%、スタートアップなら -20〜40%。

### 運用費の参考レンジ（AI 推論ベースのシステム）

装置 50 台規模・5 年保管・バッチ集約前提：

| 規模・特性 | クラウド月額 | オンプレ月額 |
|---|---|---|
| 軽量（時系列のみ） | 5〜15 万円 | 3〜6 万円 |
| 中規模（推論あり） | 10〜25 万円 | 6〜13 万円 |
| 重量級（GPU 必要） | 15〜35 万円 | 9〜19 万円 |
| LLM API 込み | 13〜35 万円 | 11〜33 万円 |

### 効果項目の参考レンジ

| 効果カテゴリ | 改善率の目安 | 備考 |
|---|---|---|
| 工数削減 | 20〜50% | 自動化の範囲次第 |
| 不良率低減 | 5〜20% | プロセス系の効果 |
| 計画外停止削減 | 30〜50% | 予知保全の効果 |
| 装置稼働率向上 | 5〜15% | 連携効果 |

**注**：すべて推測ベース。客先実態に応じて要調整。

## 整合性のチェックポイント

複数資料を作成する案件で**必ず確認**：

### 提案書 ↔ ROI 試算 Excel
- [ ] 提案書の「概算金額感」 = ROI Excel の「投資コスト」 が一致
- [ ] 提案書の「期待される効果」 = ROI Excel の「効果項目」 が対応
- [ ] 提案書の「年間運用保守」 = ROI Excel の「年間運用保守」 が一致
- [ ] フェーズ・段階の命名が複数資料間で共通（命名は案件ごとに自由、ただし資料間で一貫）

### 提案書 ↔ 運用費試算スライド
- [ ] 提案書の運用フェーズ金額 = 運用費試算スライドの月額 × 12 が一致
- [ ] 構成（クラウド / オンプレ）の前提が共通

### ROI Excel ↔ 運用費試算スライド
- [ ] 年間運用保守の数字が一致（or 内訳として整合）
- [ ] 装置数・規模の前提が共通
- [ ] 構成タイプ（クラウド / オンプレ）の選択が共通

### 全資料共通
- [ ] 客先固有名は全資料で抽象化されているか
- [ ] 「貴社」「弊社」「担当の方」 が統一されているか
- [ ] 禁止語（PoC / ML 等）が残っていないか
- [ ] フェーズ・段階の命名が資料間で一貫しているか

## 文書タイプ追加 ── 進め方・スケジュール

### 進め方スライド（4 STEP 構造）

提案書の最終ページで「次のステップ」 を示す。

#### 標準構成
- タイトル：「進め方・次のステップ」 or 「5. 進め方」
- リード文：「以下の 4 STEP で段階的に進めます」
- 4 STEP のカード横並び
  - STEP 1: 要件定義（約 2 ヶ月）
  - STEP 2: 開発（約 3〜4 ヶ月）
  - STEP 3: 実環境検証（約 2〜3 ヶ月）
  - STEP 4: 運用・展開（継続）
- 注記：各 STEP の期間は仮置き、要件次第で調整

### スケジュール表（ガントチャート）

各フェーズの期間を視覚的に示す。

#### 標準構成
- ヘッダー：フェーズ / 主要活動 / 月（横軸）/ 区分
- 各行：フェーズ名 + 期間バー
- バーの色：「進行中」 = GRAY_MID、「本提案範囲」 = MAGENTA、「概算範囲」 = NAVY
- 注記：日程は目安、実態次第で変動

## 文書タイプ追加 ── 業務フロー比較

提案の Before / After を業務フローで対比。

#### 標準構成
- タイトル：「業務フローの変化」 or 「導入前後の比較」
- 2 行レイアウト
  - 【現状】（上）：グレー系のボックス、矢印で繋ぐ
  - 【提案後】（下）：NAVY 系のボックス、AI 部分を強調

#### ボックスの内容
- 各 step の業務名（例：「受付」「目視確認」「手入力」「保管」）
- 短い動詞句で

#### 色の使い分け
- 現状：GRAY_BG 背景 + GRAY_MID 枠線（控えめ）
- 提案後：NAVY_LIGHT 背景 + NAVY 枠線（強調）
- AI が入る step：MAGENTA で強調

## 提案書の章立て（汎用テンプレ）

新規案件で「**どう章立てするか**」 のテンプレ。業界・案件に応じて調整。

### パターン A：3〜4 つの個別提案を持つ案件

```
0. 表紙
1. 背景と過去の検討
   1.1 業務フロー（現状）
   1.2 過去の検討経緯
2. 本提案の方向性
3. ご提案
   3.1 提案 1 ─ ◯◯
   3.2 提案 2 ─ ××
   3.3 提案 3 ─ △△
4. 概算金額感
5. Appendix（今後の展開候補）
6. 進め方・次のステップ
```

### パターン B：単一テーマ深掘り案件

```
0. 表紙
1. 背景
   1.1 業界背景
   1.2 課題
   1.3 過去の検討
2. 本提案の方向性
   2.1 全体アプローチ
   2.2 サービス全体像
   2.3 適用業務
3. システムの内容
   3.1 入力
   3.2 AI 処理
   3.3 出力
   3.4 期待される効果
   3.5 業務フロー（Before/After）
   3.6 UI イメージ
4. 実行計画と見積
   4.1 プロジェクトスケジュール
   4.2 実施体制
   4.3 見積
```

### パターン C：複数テーマ並行案件

```
0. 表紙
1. 全体提案
   1.1 4 テーマの構造（テーマ間連携）
   1.2 推奨優先度
2. テーマ別詳細（各テーマで 1 つの提案書 + パーツ資料）
   - 01_XXX_v1.pptx
   - 02_YYY_v1.pptx
   - 03_ZZZ_v1.pptx
   - 04_WWW_v1.pptx
3. 共通資料
   - 入出力整理（4 テーマ分）
   - 運用費試算（4 テーマ分）
   - ROI 試算 Excel
```

### どのパターンを選ぶか
- **A**：1 つの案件に複数提案がある場合（典型的な営業提案書）
- **B**：1 つのテーマを深く提案する場合（単一テーマ提案書）
- **C**：複数テーマを並行する大型案件（複数テーマ提案）
