# 価格戦略

## 概要

価格設定は、事業における最も影響力の高い戦略レバーのひとつです。価格をわずかに動かすだけで、その差はほぼそのまま利益に直結します。しかし多くの企業は、顧客が実際に何を価値として認めているかを検証しないまま、コスト積み上げ方式に頼りがちです。本フレームワークは、価格を顧客への提供価値に根ざして決定し、採算性と競合ポジションをあらかじめ検証したうえで価格構造を確定するための、規律ある進め方を提供します。

## 使用する場面

- 新製品・新サービスのローンチ価格を検討しており、コスト積み上げ・市場相場・価値捕捉のいずれで設定するか議論している。
- 既存価格が競合他社・顧客・マージン目標からのプレッシャーにさらされており、場当たり的な対応ではなく構造的な検討が必要。
- 段階的なティア設定・バンドル・数量割引の追加を検討しており、ローンチ前にマージンとミックスへの影響を把握したい。
- プレミアム・ポジショニングの採用を検討しており、貴社の支払意欲がそれを支えうるかを確認したい。

**使用しない場面：**

- 取引所で公開価格が決まるコモディティ製品（価格は外部から決定される。別の市場戦略フレームワークを適用する）。
- 規制・政府調達により価格が支配される案件（価値ロジックではなく、調達規則が価格を決める）。
- 事業全体の ROI モデルが必要な場合は `strat-business-case` を使用する。価格設定はその入力のひとつとして扱う。

## 必要な情報

フレームワークを適用する前に、以下を整理してください。不明な項目はユーザーに確認を求めます。数値は勝手に作成しません。

1. **製品・サービスの概要** — 何を価格設定するのか、対象顧客は誰か、どの課題を解決するのか。
2. **現在または想定のコスト構造** — 単位当たり変動費、関連する固定費配賦、価格下限（これ以下には設定できない水準）。
3. **競合の参照価格** — 顧客が現在、代替手段（直接競合・代替品・現状維持）に支払っている金額。
4. **顧客価値の指標** — 顧客が得られる定量的な便益（削減できる工数・増加する売上・削減できるコスト・低減できるリスク）。入手できない場合は、推計値をユーザーに確認してから進める。
5. **対象顧客セグメント** — 購入者は誰で、予算感度はどの程度か。セグメント間で支払意欲に大きな差があるか。
6. **販売量と構成の想定** — 価格帯ごとの想定販売数量、または価格弾力性に関する既存データ。
7. **戦略的意図** — マージン最大化・市場浸透・プレミアム・ポジション確立・バンドル戦略のどれを目指すのか。

## 進め方

1. **主軸となる価格ロジックを選択する。** コスト積み上げ（変動費を基準にした価格下限；規制業種・コモディティに適する）・バリューベース（顧客への定量的価値を基準；マージン最大化の余地が最も大きい）・競合基準（顧客がすでに参照している競合価格に対して設定）の3方式を比較する。複数を組み合わせる場合も、主軸となるロジックをひとつ明示し、その理由を示す。

2. **価値と支払意欲を推計する。** 顧客が次善の代替手段と比較して本製品から得られる経済的価値を定量化する。便益を「ハード（測定可能。例：年間コスト削減額）」と「ソフト（定量化が難しい。例：リスク低減・ブランド認知）」に分けて整理する。完全な情報を持つ購入者が合理的に支払う上限（バリューベースの天井）を導出する。次に、顧客の不確実性・切替えコスト・情報の非対称性を考慮した割引を適用し、推定支払意欲を算出する。顧客調査やコンジョイント分析のデータがあれば活用し、なければ推計である旨を明示する。

3. **価格構造を設計する。** 単一価格・ティア（松竹梅など）・バンドル・従量課金・サブスクリプションのいずれが最適かを判断する。構造はセグメントごとの価値享受の仕方を反映すべきであり、支払意欲の高いセグメントが上位ティアへ自然に移行できる設計にする（同一の内容に割増料金を払うだけの構造は避ける）。割引ポリシー・価格下限・ガードレール（例：変動費以下の取引は禁止）を明記する。

4. **マージンと販売量への影響を試算する。** 提案する価格帯・ティアごとに、貢献マージン（価格から変動費を引いた額）を計算し、実現可能な販売量を試算する。価格と販売量のトレードオフを分析する。すなわち、値上げが全体貢献額を毀損しない範囲での販売数の減少許容幅と、低価格で同等の貢献額を維持するために必要な販売量を明確にする。推計値はユーザーへの確認を求める。

5. **推奨案とガードレールを示す。** 価格モデル・具体的な価格帯またはティア構造・根拠（価値・マージン分析に基づく）・推奨が変わる条件を明確に示す。ガードレールを設定する。最低許容マージン・競合動向による見直しトリガー・次に収集すべきデータ（例：初期顧客からの価格フィードバック）を指定する。

## 成果物

このスキルを適用すると、以下の形式のスタンドアロン Markdown 分析を生成します。必須タイトルから始め、各セクションを順に展開し、最後を「要点」と「確認事項」で締めます。

```
# 価格戦略 — <対象>

## 価格ロジック
## 価値と支払意欲
## 価格構造
## マージンと販売量への影響
## 推奨案
## 要点
## 確認事項
```

タイトルは `# 価格戦略 — <対象>` の形式に従います。`<対象>` は価格設定する具体的な製品・サービス・案件を指します。分析は必ず `## 要点`（最重要の結論を箇条書きで簡潔に）で締め、その後に `## 確認事項`（貴社に確認が必要な事項・追加データ・後続の意思決定が必要な項目を列挙）を続けます。数値は勝手に作成せず、不明な数量は確認事項として残します。

## 例

**例 — 架空の国内 B2B SaaS 企業「ロジプレディクト株式会社」（例示・確認前）**

ロジプレディクト株式会社は、中堅製造業向けの在庫需要予測 SaaS を提供しています。現在はユーザー数に応じた月額定額制（コスト積み上げを基準に設定）を採用しています。しかし、顧客が享受している価値に対してマージンが低すぎる可能性があると感じています。

**価格ロジック（例示）：** チームは、バリューベース価格設定を主軸とすることを決めます。ロジプレディクトのツールは顧客の過剰在庫を定量的に削減できます。これはコスト積み上げ方式では見落とされる、測定可能な便益です。

**価値と支払意欲（例示）：** 試算対象の中堅顧客は、在庫評価額がおよそ X 億円規模とします（例示・確認前）。ツールの導入により、過剰在庫が推定 Y% 削減され、年間 Z 万円の運転資本が解放される想定です（例示・確認前）。完全な情報を持つ購入者が合理的に支払うバリューベースの上限は、その削減額の一部となります。顧客の不確実性と切替えコストへの割引を適用すると、推定支払意欲は現行価格を大幅に上回ります（X・Y・Z の実際の数値は貴社による確認・提供が必要です）。

**価格構造（例示）：** 3 ティアを提案します。スタンダード（単一拠点・基本需要予測）・プロフェッショナル（複数拠点・高度な分析レポート）・エンタープライズ（カスタム連携・専任サポート）。支払意欲の高い顧客が上位ティアへ自然に移行できる設計とし、低価格ユーザーが高機能を享受するだけの構造は避けます。

**マージンと販売量への影響（例示）：** プロフェッショナルティアの提案価格において、現行の定額制と比較して貢献マージンの改善が見込まれます（例示・確認前）。損益分岐分析では、値上げが全体貢献額を毀損しない範囲での販売数の減少余地が確認されています（例示・確認前）。これらの数値は貴社のコストデータとの照合が必要です。

**推奨案（例示）：** バリューベースの 3 ティア構造への移行を推奨します。変動費以下の値引きは禁止とします。主要競合が提案プロフェッショナルティア価格の 20% 超の値下げを発表した場合は、価格見直しのトリガーとして設定します。
